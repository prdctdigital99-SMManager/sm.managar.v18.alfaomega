# Vercel Speed Insights Setup

This project has been configured with Vercel Speed Insights to monitor web performance metrics.

## What's Installed

- **Package**: `@vercel/speed-insights` v2.0.0
- **Integration**: Vanilla JavaScript implementation using ES modules

## Implementation Details

### Files Modified

1. **index.html**
   - Added Speed Insights initialization script in the `<head>` section (line 514-519)
   - The script imports and calls `injectSpeedInsights()` from the Speed Insights library

2. **package.json**
   - Added `@vercel/speed-insights` as a dependency
   - Added build script to copy Speed Insights files to dist directory
   - Updated project metadata

3. **New Files Created**
   - `.gitignore` - Excludes node_modules from git
   - `vercel.json` - Vercel deployment configuration
   - `dist/speed-insights/` - Speed Insights library files (generated by build script)

## How It Works

The Speed Insights library is automatically loaded when users visit the site. It:
- Collects Core Web Vitals metrics (LCP, FID, CLS, TTFB, FCP)
- Sends performance data to Vercel's analytics dashboard
- Works seamlessly in production when deployed to Vercel

## Build Process

Before deployment, run:
```bash
npm install
npm run build
```

This will:
1. Install dependencies including `@vercel/speed-insights`
2. Copy the Speed Insights library files to the `dist/` directory

## Deployment

When deploying to Vercel:
1. The build command will automatically run
2. Speed Insights will be enabled once you activate it in your Vercel dashboard
3. Navigate to your project in Vercel Dashboard → Speed Insights → Enable

## Viewing Analytics

After deployment and some user visits:
1. Go to your Vercel project dashboard
2. Click on "Speed Insights" in the sidebar
3. View real-time performance metrics and trends

## Development

For local development with a server:
```bash
npm run dev
```

This starts a local HTTP server on port 8000.

## Notes

- Speed Insights data is only sent in production environments
- The library automatically detects the environment
- No API keys or configuration needed for Vercel-hosted projects
- For self-hosting, you may need to configure the `dsn` option
